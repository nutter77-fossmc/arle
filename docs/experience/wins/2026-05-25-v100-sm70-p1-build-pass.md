# V100 sm_70 P1 build pass — Tesla V100-SXM2-32GB, 2026-05-25

## Context

ARLE pre-this-session: `crates/cuda-kernels/build.rs` hard-rejects sm_70 at
build time per `docs/plans/sm-coverage.md` T3 policy ("Most kernels assume
native bf16, only available from sm_80"). V100 was the canonical "T3 unsupported"
exemplar — no published path to compile, let alone run, on Volta.

Goal: get `cargo check -p infer --features cuda` green on V100 (Tesla
V100-SXM2-32GB, sm_70, CUDA 12.4) without regressing T1 (A100/L4/H100), using
**per-kernel operator-level dispatch** rather than tearing the top-level T3
reject open. Validation environment: ssh host `v100` (`~/agent-infer-v100-audit`
clean checkout, `~/tilelang-sm70-copy` for the patched upstream build).

Co-driven: Claude direction + review, codex (tmux 0:1) execution.

## What worked

### Layer 1 — TileLang upstream fix (PR #2257)

V100 BF16 paged attention failed at TileLang lowering with `Layout infer conflict
between p and p_bf16 in T.Parallel loop` before any nvcc call. Root cause:
`T.copy(fragment → fragment)` SIMT loop bound two incompatible fragment layouts
on sm_70 (Volta MMA-A load layout vs softmax/parallel write layout). The same
kernels emit cubin cleanly on sm_89.

Two-cut patch landed in `src/backend/cuda/op/copy.cc` + Volta MMA path:

1. **`NeedsVoltaFragmentStaging`** predicate fires when `TargetIsVolta(target)
   && IsFragmentBuffer(op.src) && IsFragmentBuffer(op.dst) && op.src->dtype !=
   op.dst->dtype`. Lowering routes such copies through a shared-memory staging
   buffer with `tvm_storage_sync("shared")` between the two `LowerNormalCopy`
   halves, breaking the layout-coupling constraint.
2. **Volta BF16 GEMM fallback** — `mma_sm70.h` `kBFloat16, kBFloat16 →
   kFloat32` static assertion replaced with FP16-MMA fallback: BF16 fragments
   value-cast to FP16 inputs, FP32 accumulation preserved.

Gates triggered only when `TargetIsVolta(target)`. T1 (sm_80+) paths untouched
by construction.

Regression test `testing/python/kernel/test_tilelang_kernel_sm70_fragment_copy.py`
mirrors ARLE's `softmax p → p_bf16 → P@V` failure mode with a minimal 64×16
in-component kernel — no ARLE coupling. 2/2 pytest passed on V100 in 42.48s.

PR: <https://github.com/tile-ai/tilelang/pull/2257>. Branch
`fix/sm70-volta-fragment-copy` cut clean from `upstream/main` (HEAD a84c9b14),
single-file fix `src/backend/cuda/op/copy.cc +52 -0` plus the test, no
contamination from the local `metal-gemm-scalar-fallback` working branch's 19
unrelated files / 940 lines.

### Layer 2 — ARLE build.rs sm_70 admission + per-kernel cubin filter

`crates/cuda-kernels/build.rs` extended with a `T0-legacy` tier alongside
T1/T2 and a `has_legacy_volta()` predicate. AOT cubin emission gated per kernel
via a new `allow_sm70: bool` field on each `TileLangKernelSpec`, computed by:

```rust
fn sm70_qwen35_dense_head_config(q: u32, kv: u32) -> bool {
    matches!((q, kv), (32, 8) | (40, 8))
}
```

Narrows sm_70 cubins to exactly the Qwen3.5-4B (q=32, kv=8) and Qwen3.5-9B
(q=40, kv=8) dense paged-attention shapes. FP8 / DSv4 / GDR / non-P1 head
configs return `CUDA_ERROR_NOT_SUPPORTED` at runtime per the cargo:warning
banner emitted by build.rs. Marlin kernels (`marlin_kernel`, `marlin_repack`,
`marlin_w4a8_kernel`) compile under `-DARLE_DISABLE_MARLIN_SM70=1` so the
sm_70 fat-binary section is empty for them. `disable_marlin_w4_fp8` auto-true
on Volta.

`gen_tilelang_aot.py::parse_target` migrated string-form `"cuda -arch=sm_70"`
to dict-form `{"kind": "cuda", "arch": "sm_70"}` to match TileLang main HEAD's
Target API. Probe in `build.rs` updated to enumerate both editable-install
layouts (`pkg/src/tl_templates` and `pkg.parent/src/tl_templates`) instead of
single-path probe.

`docs/plans/sm-coverage.md`, `docs/support-matrix.md`, and
`crates/cuda-kernels/AGENTS.md` updated to declare the T0-legacy tier and
document its SM-pinned Qwen3.5-dense-only scope.

### Layer 3 — V100 build green

`ssh v100 cargo check -p infer --features cuda` (with
`TORCH_CUDA_ARCH_LIST=7.0 TVM_FFI_DISABLE_TORCH_C_DLPACK=1
INFER_TILELANG_PYTHON=/tmp/arle_tilelang_python.sh`) finished in **6m19s**, 0
errors, 3 warnings. Native CUDA compile, TileLang AOT (9 sm_70 cubins emitted
including the four P1 SKUs `batch_decode_paged_hd128_q32_kv8_sm70` /
`q40_kv8_sm70` and `batch_prefill_paged_hd128_q32_kv8_sm70` / `q40_kv8_sm70`),
Rust typecheck — all green.

## Rule

- **TileLang BF16 paged attention on Volta requires the upstream PR #2257
  fragment-staging + BF16-MMA fallback patches.** Without them, sm_70 lowering
  fails at `parallel.cc:589` before nvcc is invoked. Track upstream merge; if
  it drifts, vendor the patch via `crates/cuda-kernels/tools/tilelang/vendor/`
  or pin TileLang to a fork sha.
- **`T0-legacy` tier opt-in:** sm_70 is never in the default fat-binary set.
  Build it with `TORCH_CUDA_ARCH_LIST="7.0"` explicitly. T1 default
  `{80, 86, 89, 90}` is unchanged.
- **Per-kernel `allow_sm70: bool` is the canonical scope knob.** New TileLang
  AOT kernels default to `false` on sm_70; opt in only after verifying the
  head config emits cleanly under the patched TileLang.
- **DSv4-Flash, FP8 KV, INT8 split-KV attention, GDR (linear-attn), Marlin
  variants are explicit `not-supported` on sm_70 by build-time exclusion.**
  Runtime calls return `CUDA_ERROR_NOT_SUPPORTED` rather than segfault.
