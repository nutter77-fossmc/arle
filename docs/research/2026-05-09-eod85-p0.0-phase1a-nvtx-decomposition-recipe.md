## §0 SOLID gates

### Gate 1 — first decode 隔离

**Issue**:`step_decode_kernel_launch` covers ALL decode steps(first +
steady-state)。For TTFT decomposition,we want only **first** decode after
prefill。

**Workaround**(简单):
- Multi-tenant burst is dominated by first-decode-after-prefill latency
  because `bench_multitenant_burst.py` issues short-output(200 token)
  requests
- Total `step_decode_kernel_launch` time / requests ≈ first-decode
  contribution(approximately)
- 不完美 but enough for 4-phase ROI ranking

**More accurate option**:
- Add `step_first_decode_after_prefill` sub-scope(needs scheduler logic
  to detect "first decode after prefill batch landed")— ~30 LOC
- Defer to Phase 1.B if Step 4 results are ambiguous

### Gate 2 — wall-clock framing

**§0 rule**:nsys per-NVTX-window framing trap(2026-05-08 EOD+19)。
**MUST** report:
- Per-request absolute ms(scope total / N_requests)
- TTFT per request:scope ms / per-request TTFT(% of total time
  contributed by phase)

**NOT**:scope total / nsys window total(window framing,can over-claim
if nsys captures decode-heavy section)

### Gate 3 — N controlled

**Run protocol**:
- N=5 paired runs(matching `3c334ef` LICENSE protocol)
- σ/mean < 5% required for SOLID
- If σ/mean > 10% on any phase → re-run with 60s duration or larger N

### Gate 4 — SGLang baseline 同条件

Per EOD+83 brief negative case:**re-bench SGLang too**(or verify
baseline still holds)。Otherwise comparing post-2026-05-07 ARLE vs
2026-05-07 SGLang = stale-baseline trap。
