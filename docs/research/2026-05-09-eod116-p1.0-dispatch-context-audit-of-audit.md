## Anti-pattern #25 prevention applied

Per R4 #6 KILL lesson(`fe9ea8a` + `3b9cc06`):override fired in PREFILL when
target was DECODE → bench revealed +37% ITL regression。

**P1.0 design avoids this**:
- Decode context EXPLICITLY routes hybrid to MarlinW4Gemm(W4A16 path)
- Prefill context EXPLICITLY gated by `batch > 1` + env var + alignment check
- No accidental override leak into untested batch range
- Implementation context **strict subset** of hypothesis target(batch>1 prefill)

Compare R4 #6 implementation:`if batch > 1 && marlin_prefill_aligned()`
unconditional override。Implementation context = `batch > 1`(includes prefill
batch=4096 + decode-batched batch=2-8)。Hypothesis target = decode M ≤ 8。
Implementation context **broader** than target → leak → KILL。

P1.0 implementation:`batch > 1 && hybrid_w4a8_prefill_enabled() && aligned`
+ explicit decode-path branch。Implementation context = strictly prefill +
explicit opt-in。Hypothesis target = hybrid prefill。Match。✓
